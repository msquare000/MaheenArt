│                                                                                │

│  ┌────────────────┐            ┌────────────────┐                              │

│  │   │ flips  │   │            │ ▪   ▪    ▪   ▪ │                              │

│  └────────────────┘            └────────────────┘                              │
│  name reveals, one             tiles scattered on a                             │
│  centered picture              jittered grid, all visible                       │
│  flips every 700ms                                                              │
│                                                                                │
│ ③ FLY (4.2 → 5.8s)            ④ ORBIT (forever)                                │
│  ┌────────────────┐            ┌────────────────┐                              │
│  │ ▪  ╲      ╱  ▪ │            │        ▪▪▪▪    │  right reel (upper-right)    │
│  │   ╲  ╲  ╱  ╱   │            │      ▪▪    ▪   │   tilted ellipse, spinning   │
│  │ ▪──▶ ╲╳╱  ◀──▪ │  ══════▶   │  Stillness,    │  ← text centered ON TOP      │
│  │   ╱  ╱  ╲  ╲   │            │   ▪   painted  │    (+ soft scrim behind)     │
│  │ ▪  ╱      ╲  ▪ │            │  ▪▪▪▪          │  left reel (lower-left)      │
│  └────────────────┘            └────────────────┘   counter-rotating           │
│  each tile glides from its     two tilted ellipses along the NW–SE diagonal    │
│  scatter spot into its reel    keep spinning; tiles scale/fade by depth        │
│                                                                                │
│ HOW THE HANDOFF STAYS SMOOTH (the key trick)                                    │
│   Never "hand over" the transform between GSAP and the loop — that causes a     │
│   visible jump. Instead each tile keeps BOTH positions and a blend value:       │
│                                                                                │
│     scatterPos ──────── blend 0…1 ────────▶ orbitPos(t)                        │
│                                                                                │
│   The rAF loop runs every frame and paints:                                     │
│     x = lerp(scatter.x, orbit.x, blend)      (same for y / rot / scale / opacity)│
│   GSAP only tweens `blend` 0 → 1 (staggered per tile). At blend=0 the tile sits  │
│   exactly on its scatter spot; at blend=1 it is purely orbiting. Zero jump,      │
│   and the orbit keeps running forever afterwards.                               │
│                                                                                │
│ WHAT I USED TO ACHIEVE THIS ANIMATION                                          │
│   1. GSAP timeline — sequences the 4 phases; per-tile stagger on both the       │
│        spread (0.03s) and the flight (0.04s). Ease: power3.out / power3.inOut.  │
│   2. One shared writer: applyTile() builds the SAME transform string in every   │
│        phase → `translate(-50%,-50%) translate(x,y) rotate(r) scale(s)`.        │
│        Nothing else writes transform, so phases can't fight each other.         │
│   3. PRELOAD: all tiles stacked at center at scale 2.6; only the current one is │
│        opaque; setInterval flips which one → looks like one big changing image. │
│   4. SPREAD: GSAP tweens each tile center → its scatter target, scale 2.6 → 1.  │
│        Targets = a jittered grid sized cols≈√(n·W/H), so corners get covered.   │
│   5. FLY + ORBIT: the blend trick above. orbitPos() is a TILTED ELLIPSE =       │
│        the "reel":                                                              │
│           lx = cos(a)·rx        x = cx + lx·cos(θ) − ly·sin(θ)                   │
│           ly = sin(a)·ry        y = cy + lx·sin(θ) + ly·cos(θ)                   │
│        θ = reel tilt (35°). Reel A center (−offX,+offY), reel B (+offX,−offY)   │
│        → the two reels sit on a diagonal. Mirrored spin (dir = ±1).             │
│        depth = (1−sin a)/2 drives scale, opacity and z-index → 3-D feel.        │
│   6. TEXT: centered, z-index 100 (above both reels). A radial scrim (z-90)      │
│        fades the wall behind it so the copy stays readable over the paintings.  │
│   7. Control panel: every timing, both reel tilts, diagonal offsets, radii,     │
│        period, spin direction, tile count/size, scrim on/off, replay + skip.    │
│                                                                                │
│ NOTES: Uses 24 of the 45 paintings by default (spread evenly across the set),   │
│        count adjustable 10–40. NOT yet implemented into project code — the      │
│        current hero has ONE orbit (17 items); this replaces it with TWO.        │

│        frame's height from a % of viewport and width = height*ratio (clamped   │
│        to 80vw), transitioned .55s so the frame morphs between shapes. A        │
│        circular option adds border-radius:50% to frame+screen (ring frame).    │

│   6. Control panel drives CSS vars + rebuilds the ScrollTrigger live: start/   │
